## 9. Implementation

### 9.1 Strategy

Custom strategy `interval-reminder.ts` (or reuse `interval.ts` with a "repeat indefinitely" flag).

Behavior:
- Tick: 20 minutes (configurable).
- On expiry: trigger browser notification + audio cue.
- Auto-restart on user acknowledge (or after 20 seconds for the "look away" period).
- Persist across page reloads via `localStorage` so a user closing/reopening the tab doesn't reset.

### 9.2 UI requirements

- **Minimal**: a clean readout + start/pause + notification opt-in.
- **Background-tab friendly**: title bar shows time-remaining (e.g., `(05:23) 20-20-20`).
- **Notification permission flow**: explicit "Enable browser notifications" button ŌĆö never auto-prompt.
- **Look-away phase**: when timer expires, show a 20-second countdown with "Look 20 feet away" instruction and a soft chime.
- **Configurable interval** (20/40/60 min) for users who prefer less frequent breaks.

### 9.3 Critical implementation details

- **Web Notifications API** for the alert (with audio fallback if notifications denied).
- **Page visibility API** for accurate background-tab timing (the timer must keep counting even when the tab is backgrounded).
- **localStorage persistence** of start time + interval, so reloading doesn't restart the cycle.
- **No wake-lock needed** (this runs while the user is on other tabs).
