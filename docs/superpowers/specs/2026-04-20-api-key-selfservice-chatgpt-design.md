# API Key Self-Service + ChatGPT GPT

User-facing API key management and ChatGPT custom GPT integration. Users generate their own API keys from `/developers/keys`. A ChatGPT GPT points at the public OpenAPI spec for zero-auth timer creation.

## Problem

API keys can only be generated by admins. There's no UI for it — just a backend endpoint. Users and AI platforms have no self-service path to get API keys for authenticated endpoints (challenge creation, joining). The ChatGPT ecosystem has no GoTimer GPT despite the public API being ready.

## Design

### Database: New `api_keys` Table

Migration added to `src/lib/db.ts`:

```sql
CREATE TABLE IF NOT EXISTS api_keys (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  key TEXT NOT NULL UNIQUE,
  name TEXT NOT NULL,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX IF NOT EXISTS idx_api_keys_user_id ON api_keys(user_id);
CREATE INDEX IF NOT EXISTS idx_api_keys_key ON api_keys(key);
```

Key format: `gtmr_` + 64-char hex string (matching existing convention).
Limit: 5 keys per user.

### User API Endpoints

New route: `src/app/api/user/api-keys/route.ts`

All endpoints require login via `hazo_get_auth`. No admin permissions needed — users manage their own keys.

**GET /api/user/api-keys**
- Returns current user's keys with partial key display (first 12 chars + "...")
- Response: `{ api_keys: [{ id, name, key_preview, created_at }] }`

**POST /api/user/api-keys**
- Body: `{ name: string }` (required, max 100 chars)
- Enforces 5 key limit per user
- Prevents duplicate names per user
- Returns full key once: `{ api_key: { id, name, key, created_at }, warning: "..." }`

**DELETE /api/user/api-keys?id=xxx**
- Deletes by key ID (not name, since names could collide across users)
- Only deletes keys belonging to the authenticated user
- Returns: `{ ok: true }`

### Updated API Key Validation

Modify `src/lib/api-auth.ts`:

Current behavior: checks admin keys in `settings` table.
New behavior: checks admin keys first, then user keys in `api_keys` table. Returns `user_id` when matched against a user key.

```typescript
export async function validateApiKey(request: NextRequest): Promise<{
  valid: boolean;
  key_name?: string;
  user_id?: string;  // new: set when matched against a user key
}>
```

### User Key Management Page

New page: `src/app/developers/keys/page.tsx`

Client component requiring login (redirects to `/hazo_auth/login` if not authenticated). Uses `use_hazo_auth` for auth state.

UI:
- Header: "API Keys" with subtitle explaining what keys are for
- Key count: "3 of 5 keys used"
- Key list: table with columns — Name, Key (preview), Created, Actions (delete button)
- Create button: inline form or modal with name input. On success, shows the full key with a copy-to-clipboard button and warning that it won't be shown again.
- Delete: confirmation before revoking
- Link to developer docs (`/developers`)

Styling: follows the existing developers page patterns (white cards, rounded-xl, shadow-sm, border).

### Admin API Keys Page

New page: `src/app/admin/api-keys/page.tsx`

Shows all keys across all users:
- Admin keys (from settings table): shown with "Admin" badge
- User keys (from api_keys table): shown with user email/name
- Revoke button for any key
- Uses existing admin auth pattern (`use_hazo_auth` with `admin_view_all_games` permission)

New API endpoint: `src/app/api/admin/all-api-keys/route.ts`
- GET: returns all user keys with user info (joins against hazo_auth users)
- DELETE: revoke any user key by ID

### Admin Sidebar Update

Added to the third nav group (with SEO, Analytics, etc.) after Timer Health.

### ChatGPT Custom GPT

Configuration only — no GoTimer code changes needed.

**GPT Setup:**
- Name: "GoTimer"
- Description: "Create timers, Pomodoro sessions, chess clocks, and more. Get shareable links for any timer type."
- Instructions: prompt telling the GPT how to use the API (create timers, list presets by category, generate embed codes)
- Actions: OpenAPI spec at `https://gotimer.org/api/openapi.json`
- Authentication: None (all timer endpoints are public)

**Documentation:** Create `docs/chatgpt-gpt-config.md` with:
- Step-by-step GPT builder instructions
- The system prompt to use
- How to configure the OpenAPI action
- Testing checklist
