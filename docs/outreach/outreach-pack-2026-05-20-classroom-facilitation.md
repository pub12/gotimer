## How to use this document

**Tone rule.** Every first email here is a *question*, not a pitch. The Tony Vincent reply chain is the template — the recipient should feel respected, not marketed to. The pitch (if any) happens in the *second* email, after they've replied.

**Pacing rule.** Send no more than 2 emails per evening. Wait 7-10 days. If no reply: send the one-line follow-up from Sequence A, once. Then move on.

**Customisation rule.** Every email below has `[bracketed]` slots. Fill them in with something you've actually read, watched, or noticed about that person. If you haven't, pick a different person — don't fake it. The classroom & facilitation worlds are small and a faked reference will get you discussed in private slack channels.

**Disclosure rule.** Always say you're the builder. Never use tracking pixels or shortened/utm links. Use plain `gotimer.org/...` URLs.

**Tracking.** Append each send to `docs/outreach/contacts.csv` using the schema from Sequence A.

**Sequence B specific lesson from Sequence A.** Tony Vincent worked because the email asked "which of these tools is *not* needed." That framing — inviting them to delete something — disarms the "is this another pitch" filter. Use the same shape in this pack: ask which tool they'd kill, which feature they'd cut, which assumption is wrong. Don't ask "what do you think?"
